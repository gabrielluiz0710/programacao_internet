Elementos e Atributos

Elemento tem o seguinte formato:

<nomeElemento>Conteúdo</nomeElemento>

Elementos podem ter vários atributos:

  • <img> pode ter src, alt
  • <input> pode ter type, disabled (que também é um exemplo de atributo vazio)

Elementos textuais (Além de p e h1...)

  • <abbr>: é usado para abreviações. Ex: HTML
  • <address>: para informações de contato. Ex:
    Rua Exemplo, 123, Bairro Centro
  • <pre>: para citações especiais de manutenção de espaços e quebras de linha. Ex: 
    Este    texto mantém    os espaços.
  • <br>: serve para quebra de linha (deve ser evitado). Ex: Linha 1
    Linha 2
  • <q>: citações curtas, apresentada com aspas. Ex: Como diz o ditado, água mole em pedra dura, tanto bate até que fura.
  • <blockquote>: citações longas. Ex:
    O homem é o lobo do homem.
  • <cite>: citações de títulos de trabalhos criativos (livros, filmes). Ex: Meu livro favorito é O Guia do Mochileiro das Galáxias.
  • <hr>: para mudar temática de parágrafo, cria uma linha horizontal.
  • <b>: negrito, porém quando não se deseja ter forte importância (uso obsoleto)
  • <strong>: negrito com forte importância
  • <i>: itálico
  • <em>: enfatização textual

Listas

  • <ul> é uma unordered list
    • lista não ordenada
      • Item 1
      • Item 2
  • <li> é um item dentro de cada lista
  1. <ol> é uma ordered list
    1. lista ordenada (com numeração)
      1. Primeiro item
      2. Segundo item
  2. <li> também adiciona itens na listas

CSS - Folha de Estilo em Cascata

Podemos inserir o CSS de três formas:

  • Embutido na linha, com o atributo style no HTML.
  • Folha de estilo embutida no HTML interno (com a tag <style> no <head>).
  • Folha de estilo externa, com elemento <link> para referenciar.
    • Melhor separação e pode ser replicada para várias páginas.
    • <link rel="stylesheet" href="style.css">

Regra: indica qual elemento será afeitado pelo CSS.

Propriedade: qual aspecto de apresentação será ajustado.

Valor: valor da propriedade.


<style>
    p {
        font-size: 14pt;
        color: blue;
    }
</style>

Seletores

  • Seletor de elemento: p, h1
  • ID: #idDoElemento
  • Filho (X > Y): Afeta apenas os Y que são filhos diretos de X. Ex: li > a
  • Descendente (X Y): Afeta todos os Y que estão dentro de X. Ex: div a
  • Irmão adjacente (X + Y): Afeta todo Y que aparece imediatamente após X. Ex: h1 + p
  • Irmão em geral (X ~ Y): Afeta todo Y que é irmão e aparece após X. Ex: h1 ~ p
  • Atributo (X[atributo]): Seleciona elementos de acordo com uma condição de seus atributos. Ex: input[required]
  • Classe (.nomeClasse):
    • Aplicar o estilo mais de uma vez.
    • Pode ser aplicado apenas a um tipo de elemento: p.destacado (só afetará os parágrafos com a classe "destacado").
    • Combinar seletores com classes: .listaHorizontal > li (apenas os <li> que são filhos diretos de elementos com a classe "listaHorizontal").
    • Um elemento pode ter mais de uma classe.

Pseudo-Classe

Altera o estilo de um elemento caso ele esteja em um estado particular.

  • :link → Estilo inicial do link.
  • :visited → Estilo de links já visitados.
  • :hover → Estilo do elemento quando o mouse passa sobre ele.
  • :active → Estilo do elemento quando ativado (clicado).
  • :valid → Estilo de campos de formulário com conteúdo válido.
  • :invalid → Estilo de campos de formulário com conteúdo inválido.
  • :checked → Campos de rádio, checkbox ou option quando selecionados.
  • :focus → Quando o elemento está em foco (edição).
  • :first-child → O primeiro filho do elemento pai. Ex: li:first-child
  • :last-child → O último filho do elemento pai.
  • :nth-child(n) → O n-ésimo filho do elemento pai.
  • :first-of-type → A primeira ocorrência do elemento dentro do seu container.
  • :has() → Selecionar elementos que contenham outros elementos associados.
    • div:has(img) → Seleciona uma <div> que tenha uma <img> como descendente.
    • div:has(> img) → Seleciona uma <div> que tenha uma <img> como filho direto.
    • h1:has(+ p) → Seleciona um <h1> que tenha um <p> imediatamente depois.
  • :is() → Recebe uma lista de seletores como argumento e seleciona os elementos que correspondem a qualquer seletor da lista.
    • :is(section, article, div):has(img) → Seleciona <section>, <article> e <div> que tenham uma imagem como descendente.

Pseudo-Elementos

Seleciona uma parte específica de um elemento (elemento::valor).

  • p::first-line → A primeira linha de cada parágrafo.
  • p::selection → O texto selecionado no parágrafo pelo usuário. (Nota: a sintaxe correta é ::selection)
  • input::placeholder → O texto de placeholder dos inputs.
  • ::after → Espaço para conteúdo localizado depois do conteúdo normal do elemento. Ex: p::after { content: 'exemplo after'; }
  • ::before → Semelhante ao ::after, mas insere conteúdo antes.

Textos

Fonte

  • font-family: Define a família da fonte (ex: Arial, Times New Roman).
  • font-size: Define o tamanho da fonte.
  • font-style: Define o estilo (italic, oblique, normal).
  • font-weight: Define a espessura (bold, normal, 100-900).
  • font-stretch: Expande ou comprime a fonte.
  • font-variant: Define variantes (small-caps, etc).
  • line-height: Define o espaçamento entre as linhas.

Ajuste de Texto

  • text-align: Alinhamento horizontal (center, justify, right, left).
  • vertical-align: Alinhamento vertical (top, middle, bottom).
  • text-decoration: Decoração do texto (none, underline, line-through).
  • text-indent: Recuo da primeira linha.
  • text-transform: Transformação de caixa (uppercase, lowercase, capitalize).
  • color: Cor do texto.

Unidades de Tamanho

  • px: Tamanho absoluto (pixels).
  • em: Relativo ao tamanho da fonte do elemento corrente (herdada do pai).
  • rem: Relativo ao tamanho da fonte do elemento raiz (<html>).
  • %: Relativo ao elemento pai.
  • vh: Relativo a 1% da altura do viewport (área visível da janela).
  • vw: Relativo a 1% da largura do viewport.
  • ch: Relativo à largura do caractere "0" da fonte do elemento.

Box Model

Margens e Paddings

margin e padding podem receber de um a quatro valores: 


/* 4 valores: superior, direita, inferior, esquerda */
margin: 20px 50px 80px 100px; 

/* 3 valores: superior, laterais (direita/esquerda), inferior */
margin: 20px 40px 20px; 

/* 2 valores: superior/inferior, laterais (direita/esquerda) */
margin: 20px 40px; 

/* 1 valor: todas as 4 direções */
margin: 20px;
  • Para ajustes individuais, podemos usar: margin-left, margin-right, margin-top, margin-bottom. O mesmo se aplica ao padding.
  • Margens verticais entre dois elementos vizinhos não se somam, prevalece a maior delas (colapso de margens).

Bordas

  • border: espessura estilo cor; (Ex: border: 1px solid red;)
  • Estilos: solid, dotted, dashed, double, none.
  • É possível ajustar cada borda individualmente (border-right, border-left, etc).
  • Aspectos específicos das bordas: border-color, border-style, border-width.
  • border-radius: Permite arredondar os cantos das bordas. O valor indica o raio da curva. Aceita até quatro valores (canto superior-esquerdo, superior-direito, inferior-direito, inferior-esquerdo).

Box-sizing

Altera o modo como a largura e a altura de um elemento são calculadas.

  • box-sizing: content-box; (valor padrão)

  • box-sizing: border-box;
    • width e height incluem os tamanhos das bordas e paddings (mas não as margens).

Exibição e Posicionamento

Display

  • none: Oculta completamente o elemento, removendo-o do fluxo da página.
  • block: Exibição em nível de bloco (como <div>). Começa em uma nova linha e ocupa toda a largura disponível.
  • inline: Exibição em linha (como <span>). Só ocupa o espaço necessário para o seu conteúdo. width, height, margin-top/bottom e padding-top/bottom não têm efeito.
  • inline-block: Comportamento em linha, mas com a possibilidade de usar width, height e margens/paddings verticais.
  • flex, grid, inline-flex, inline-grid: Para layouts complexos (Flexbox e Grid).

Visibility

  • hidden: Oculta o elemento, mas ele continua ocupando seu espaço no layout.
  • visible: Exibição normal (padrão).
  • collapse: Usado principalmente para ocultar linhas e colunas de tabelas sem alterar o layout geral.

Float

Posiciona o elemento à esquerda ou à direita de seu container, permitindo que o texto e outros elementos fluam ao seu redor.

[Aqui havia uma imagem ilustrando o Float]

Overflow

Define como o conteúdo que extrapola os limites de um elemento será apresentado.

  • Pode ser especificado para os eixos X e Y (overflow-x, overflow-y).
  • visible: O conteúdo é sempre visível, mesmo que "vaze" para fora da caixa.
  • hidden: O conteúdo que "vaza" é cortado e fica invisível.
  • scroll: Barras de rolagem são sempre apresentadas, mesmo que não sejam necessárias.
  • auto: Barras de rolagem são apresentadas apenas quando o conteúdo excede o tamanho da caixa.

Position

  • static: Padrão. O elemento segue o fluxo normal do documento.
  • relative: Relativa à sua posição normal. Pode ser deslocado com top, left, right, bottom sem afetar a posição dos elementos ao redor.
  • absolute: Removido do fluxo normal e posicionado sobre o restante do conteúdo. Sua posição é relativa ao ancestral posicionado mais próximo (ou ao <html>).
  • fixed: Posicionado com relação ao viewport (a janela do navegador). Não se move com a rolagem da página.
  • sticky: Comportamento misto. Age como relative até que um ponto de rolagem seja atingido, e então "gruda" na tela como se fosse fixed.

Transform

Permite mover, rotacionar, torcer e escalonar um elemento. Utilizado, por exemplo, para centralizar um modal com transform: translate(-50%, -50%);.

Cascade, Especificidade e Herança

  • Cascade (Cascata): Se houver declarações repetidas, a que foi declarada por último no código prevalece.
  • Especificidade: Propriedades dentro de regras mais específicas têm maior peso. Por exemplo, um seletor de ID (#meuId) é mais específico que um seletor de classe (.minhaClasse), que por sua vez é mais específico que um seletor de tag (p).
  • Herança (Inheritance): Elementos filhos herdam algumas propriedades de seus elementos pais (como font-family e color), mas não todas (width, margin, padding e border não são herdados). É possível forçar a herança com o valor inherit.

Flexbox e Responsividade

  • Capacidade do website de se ajustar para todos os dispositivos.
  • Tag de responsividade adicionada ao <head>:
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
    • O conteúdo expande de acordo com o tamanho do viewport.
    • A página fica melhor escalonada.

Media Queries

É possível aplicar estilos CSS apenas quando o dispositivo atende a certas condições (como largura mínima ou máxima, orientação, etc.). 


@media MediaType AND (MediaCondition) {
    /* Código CSS */
}

MediaType → opcional

  • screen: Dispositivos com tela.
  • print: Para impressão (ativado com Ctrl+P).
  • all: Todos os tipos de mídia (padrão).

MediaCondition → condição a ser satisfeita para aplicar o CSS.

  • min-width
  • max-width
  • (min-width: 400px) and (max-width: 900px)
  • orientation: portrait ou landscape

Flexbox

  • Layout flexível para design responsivo e dinâmico.
  • Container pai (container flex):
    • Ativado com display: flex;
  • Filhos (itens flexíveis):
  • Ao ativar o container flexível, os <div>s filhos deixam de ocupar toda a largura e ocupam apenas a largura necessária para a exibição do seu conteúdo.
  • Eixo principal: por padrão, é o eixo horizontal (X).
  • Eixo transversal: por padrão, é o eixo vertical (Y).

Justify-Content

  • Ajusta o alinhamento dos itens no eixo principal.
  • Usado no container, não nos itens.

[Aqui havia uma imagem ilustrando as propriedades de Justify-Content: flex-start, flex-end, center, space-between, space-around, space-evenly]

Flex-direction

  • Altera a direção do eixo principal.
  • flex-direction: row; (padrão)
  • flex-direction: column; (o eixo principal se torna vertical)
  • Usada no container.
  • Outros valores: row-reverse, column-reverse.

Align-items

  • Alinha os itens dentro de um container flex no eixo transversal.
  • align-items: stretch; (padrão) → estica os itens para ocupar todo o espaço disponível no eixo transversal.
  • align-items: center;
  • align-items: flex-start; → alinhados no início do eixo transversal.
  • align-items: flex-end; → alinhados no final do eixo transversal.
  • align-items: baseline; → alinhados com base na primeira linha de seu conteúdo.

Flex-Wrap

  • Por padrão, os itens tentam ocupar uma única linha, causando overflow se necessário.
  • flex-wrap: wrap; evita o overflow, permitindo que os itens ocupem múltiplas linhas.
    • Geralmente é combinado com as propriedades de align-content.
  • flex-wrap: nowrap; (padrão) → sem quebra de linhas.

Align-Content

  • Alinha as linhas de um container flex quando há espaço extra no eixo transversal (só funciona com flex-wrap: wrap;).
  • Ao contrário do align-items que alinha os itens dentro de sua linha, o align-content alinha as próprias linhas em relação ao container.

[Aqui havia uma imagem ilustrando as propriedades de Align-Content: flex-start, flex-end, center, space-between, space-around, stretch]

Gap

  • Insere espaçamento (calhas) entre os itens flexíveis, tanto nas linhas quanto nas colunas.
  • Com gap: val1 val2;, val1 é usado para o espaçamento entre as linhas (row-gap) e val2 para o espaçamento entre as colunas (column-gap). Se apenas um valor for fornecido, ele se aplica a ambos.

Propriedades dos Itens Flexíveis

→ Align-Self

  • Alinha itens individualmente no eixo transversal, sobrepondo o valor de align-items do container.

→ flex-grow

  • Define como os itens devem expandir para ocupar o espaço disponível no eixo principal.
  • Valor padrão: 0 (não expande).

→ flex-shrink

  • Define como os itens devem encolher quando não houver espaço suficiente.
  • Valor padrão: 1 (encolhem na mesma proporção).

→ flex-basis

  • Define o tamanho inicial de um item antes que o espaço restante seja distribuído.

→ flex

  • Propriedade abreviada para definir flex-grow, flex-shrink e flex-basis de uma só vez.

JavaScript e DOM

  • Script de alto nível integrada nos navegadores
  • Prover interatividade e dinamismo
    • Programar o comportamento da página de acordo com os eventos
    • Alterar o HTML dinamicamente
    • Adicionar novos elementos dinamicamente
    • Modificar atributos e estilos
    • Fazer requisições assíncronas
    • Validar forms
  • Comumente executada no lado do cliente (front-end)
  • Pode ser embutida no HTML, com uso de <script> tanto no head, quanto no body
  • Pode ser feito um arquivo separado e linkado com:
    • <script src="arquivoJavaScript.js"></script>
    • Vantagens
      • Melhor separação
      • Mais conciso
      • Possibilidade de reutilizar
      • Podem ser mantidos em cache pelo navegador → mais agilidade

Execução

→ Durante o carregamento da página

  • Fora de funções, é executado durante o carregamento do HTML
  • Inicializações ou registra funções tratadoras de eventos

→ Execução em respostas a eventos

  • A maior parte é executada depois que a página é carregada
  • Quando o usuário interage com o site, ocorrem eventos que disparam as funções JavaScript
  • Em geral, JavaScript funciona de modo síncrono, então para começar uma nova atividade é preciso terminar a anterior
    • É possível alterar isso com algumas ferramentas

Declaração de Variáveis

→ var nomeVariável = valorInicial

  • Escopo local, declarada dentro de uma função
  • Escopo global, fora de funções
  • Pode ser redeclarada e ter valor atualizado
  • Podem ser acessadas por window (ex: window.idade)

→ let nomeVar = valorInicial

  • Escopo restrito ao bloco de código que é declarada
  • Acessada e atualizada apenas no bloco
  • Não pode ser redeclarada no mesmo bloco

→ const nomeConstante = valor

  • Semelhante a let, porém não pode ser atualizada
  • Deve ser inicializada na declaração

<script>
    const pi = 3.14;
    var soma = 0; // soma é uma variável global
    
    for (let i = 1; i <= 10; i++) {
        soma += i; // i só pode ser acessada neste for
    }
    
    if (soma > 50) {
        let k = soma + pi; // k só pode ser acessada neste bloco
        var m = k + 1; // m poderá ser acessada fora do if
        console.log(k);
    }
    
    console.log(m); // mostrará o valor de m normalmente
    // console.log(k); // erro, pois k é restrita ao 'if' acima
</script>

Operadores

→ +

  • Permite somar ou concatenar, dependendo dos operandos
    • string - concatena
    • num - soma

→ == e ===

  • == compara apenas valores
  • === compara valor e o tipo dos operandos
    • 1 == true; // retorna true, pois true é convertido para 1
    • 1 === true; // retorna false (tipos diferentes)
    • 2 == true; // retorna false
    • 10 == "10" // retorna true depois de converter a string "10" para 10
    • 10 === "10" // retorna false; (tipos diferentes)
    • 1 == "true"; // retorna false, pois não consegue converter "true" para número

Objetos

  • window
    • Objeto global
    • Representa a aba do navegador contendo a página
    • Obter informações ou realizar ações a respeito da janela
      • Obter dimensões: window.innerWidth e window.innerHeight
      • Executar ação quando página for carregada
      • Mostrar alerta: window.alert
  • navigator
    • Navegador de internet em uso
    • Fornece dados como idioma, geolocalização, memória, etc
    • Ex: window.alert(navigator.language);
  • document
    • Documento HTML carregado
    • Manipulação da árvore DOM
  • console
    • Janela console de depuração do navegador
    • console.log, console.warn, console.error

Strings

  • Strings com aspas duplas podem conter aspas simples e vice-versa
    • let msg = "It's alright";
  • Strings são objetos com propriedades e métodos
    • msg.length
    • msg.indexOf('alright')
    • msg.substr(0,2)
    • msg.split, etc
  • Caracteres podem ser acessados por colchetes ou pelo método charAt
    • let primLetra = msg[0];
    • let primLetra = msg.charAt(0);
  • São imutáveis (caracteres da string não podem ser alterados)
  • Para utilização de variáveis e expressões, é preciso utilizar Template Strings
    • Strings definidas com crase `minha string`
    • Variáveis e expressões utilizadas com ${ }

Arrays

  • Objetos, com propriedades e métodos
  • Elementos são acessados por índices numéricos
  • Os elementos são colocados entre colchetes, separados por vírgula
    • let pares = [2, 4, 6, 8];
    • let primeiroPar = pares[0]; // 1º elemento
    • let nroElementos = pares.length; // tamanho do array
    • pares[3] = 10; // altera o 4º elemento do array para 10
  • É possível ter elementos de tipos diferentes
    • let arrayMisto = [2, 'A', true];
  • O array pode ser iniciado vazio
    • let pares = [];
  • Outros métodos
    • .push() → adiciona ao final do array
    • .pop() → remove e retorna o ultimo item
    • .unshift() → adiciona um item no início
    • .shift() → remove e retorna o primeiro item

Percorrendo array com for


let pares = [2, 4, 6, 8];
for (let i = 0; i < pares.length; i++) {
    console.log(pares[i]);
};

let pares = [2, 4, 6, 8];
for (let item of pares) {
    console.log(item);
};

let pares = [2, 4, 6, 8];
for (let i in pares) {
    console.log(pares[i]);
};

Métodos

  • forEach
    • Executar uma função, passada como parâmetro, para cada elemento do array
    • Sempre retorna undefined
  • map
    • Produzir um novo array após aplicação de uma função em cada elemento do array existente

Árvore DOM

  • Representação da estrutura do documento HTML
  • Cada elemento, comentário ou texto é representado como objeto (nó)
  • Permite manipulação dinâmica utilizando DOM API e JavaScript

EventTarget: interface que permite aos nós da árvore DOM suportarem eventos. Inclui definições como addEventListener

Node: classe abstrata com definições de propriedades e métodos comuns a todos os tipos de nós. Inclui propriedades como parentNode e childNodes. Herda as definições de EventTarget

Element: interface com funcionalidades a nível de elemento. Inclui propriedades como nextElementSibling e children. Herda as definições de Node

HTMLInputElement: fornece propriedade e métodos adicionais para elementos do tipo input, como value, pattern, required etc. Herda as definições de HTMLElement

Document: interface que representa o documento HTML carregado na aba do navegador. Serve de ponto de entrada para a árvore DOM. Contém definições a nível de documento como getElementById e createElement

Busca na Árvore

  • document.querySelector("seletor CSS")
    • Retorna primeiro nó da árvore do tipo Element que atende à string
  • document.querySelectorAll("seletor CSS")
    • Aceita string como parâmetro
    • Retorna lista com todos os nós da árvore DOM que atendem
  • document.getElementById("id")
    • Busca um nó de elemento utilizando o seu id
  • document.getElementsByName("name")
    • Busca nós de elementos pelo valor do atributo name do elemento
  • document.getElementsByTagName("tag")
    • Busca nós de elementos pelo nome da tag HTML, como img, h1, etc.
  • document.getElementsByClassName("class")
    • Busca nós de elementos pelo valor do atributo class do elemento

addEventListener

  • Registrar múltiplas funções para tratar evento no objeto
  • Quando evento ocorrer, serão chamadas na ordem registradas

const nodeImage = document.querySelector("#imgUfu");
nodeImage.addEventListener('click', funcao1);
nodeImage.addEventListener('click', funcao2);

Funções Anônimas

  • Definidas no momento que são indicadas para tratar o evento

const nodeImage = document.querySelector("#imgUfu");
nodeImage.addEventListener('click', function() {
    alert("voce clicou na logo da UFU!");
});
  • ⇒ → permite criar funcoes sem a palavra function, apenas com =>
    • objeto.onclick = () => alert('Você clicou em...');

Load

  • Ocorre quando a página termina de carregar por completo
    • Baixar as imagens, css, etc

DOMContentLoaded

  • Documento é carregado e a árvore DOM montada
    • Não aguarda carregamento de outros recursos
    • Geralmente ocorre antes de load

Manipulação da Árvore DOM (Objeto Event 'e')

  • e.target

  • e.screenY

    • Coordenada x do clique na viewport

Acesso ao Conteúdo dos Elementos

  • Propriedade textContent
    • Se o elemento possui conteúdo textual, retorna esse texto acrescido da concatenação do textContent de eventuais elementos filhos
    • Uma alteração do valor removerá eventuais nós filhos e substituirá pelo novo texto
  • Propriedade innerText
    • Semelhante a textContent, porém não inclui conteúdos de elementos que “não podem ser lidos” pelo usuário, como conteúdo ocultado com CSS, conteúdo de tags.
  • Propriedade innerHTML
    • Retorna todo o conteúdo do elemento, incluindo as tags HTML
    • Quando alterada, o novo conteúdo é avaliado pelo navegador. Caso contenha tags HTML, levará à criação de novos nós correspondentes na árvore DOM

Método Element

  • Element.insertAdjacentHTML()
    • Cria e insere nós na árvore DOM a partir de string contendo código HTML
    • Permite indicar posição de inserção
    • node.insertAdjacentHTML(posicao, stringHtml)
      • posicao: beforebegin, afterbegin, beforeend, afterend

Manipulando atributos e CSS


<script>
    const nodesH1 = document.querySelectorAll("h1");
    for (let node of nodesH1){
        node.onclick = () => node.style.visibility = 'hidden';
    }
</script>
  • node.classList.add("classeCSS") para adicionar uma classe CSS ao elemento
  • node.classList.remove("classeCSS") para remover uma classe CSS do elemento
  • node.getAttribute("attr")
    • Permite acessar o valor do atributo conforme aparece no HTML
    • Atributos não padronizados devem ser acessados com getAttribute
  • node.setAttribute("attr", "valor")
    • Define o valor de um atributo
    • Se existe, atualiza o valor
    • Se não existe, cria um novo atributo com o valor
  • node.firstChild
    • Retorna o primeiro nó filho do elemento (abrange nós do tipo texto, comentário e elemento)
  • node.firstElementChild
    • Retorna o primeiro nó filho do tipo elemento
  • node.lastChild
    • Retorna o último nó filho (abrange nós do tipo texto, comentário e elemento)
  • node.lastElementChild
    • Retorna o último nó filho do tipo elemento
  • node.childNodes
    • Retorna uma lista contendo todos os nós filhos do nó (inclui nós de texto, nós de comentário e nós de elemento)
  • node.children
    • Retorna lista contendo apenas os nós filhos do tipo elemento
  • Outros
    • node.parentNode - retorna o nó pai do nó em questão
    • node.nextSibling - retorna o próximo nó irmão (nó de qualquer tipo)
    • node.nextElementSibling - retorna o próximo nó irmão do tipo elemento
    • node.previousSibling - retorna o nó irmão anterior (nó de qualquer tipo)
    • node.previousElementSibling - retorna o nó irmão anterior do tipo elemento
    • cloneNode(deep)
      • Duplica o objeto correspondente ao nó
      • Se o parâmetro deep for true, clona também os nós filhos
      • Pode ser usado para duplicar um ramo do documento HTML (o clone não é inserido automaticamente na árvore DOM)

Criar e Adicionar novos Nós na Árvore DOM

  • document.createElement("nomeDoElementoASerCriado")
    • Cria um novo nó do tipo Element
    • Ex.: let novoSpan = document.createElement("span");
  • node.appendChild(novoNo)
    • Acrescenta um nó filho no final da lista de filhos
  • node.removeChild(noFilhoASerRemovido)
    • Remove um nó filho (parâmetro) da lista de filhos
  • node.remove()
    • Remove o próprio nó da lista de filhos do nó pai

Propriedades `document`

  • document.head - acesso direto ao nó corresp. ao elemento <head>
  • document.body - acesso direto ao nó corresp. ao elemento <body>
  • document.title - acesso direto ao nó corresp. ao elemento <title>
  • document.location - objeto com URL da página. Pode ser modificado
  • document.forms - retorna coleção de todos os formulários (<form>)
  • document.images - retorna coleção de todas as imagens (<img>)
  • document.anchors - retorna coleção de todos os links (<a>)
