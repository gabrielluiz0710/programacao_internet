O que mudou?
O corpo da resposta antes estava vazio, agora contém uma mensagem detalhada do erro de PHP, 
além do código de status que antes era 500 Internal Server Error, agora é 200 OK.

Qual foi o código de status retornado?
200 OK

Por quê? 
Com a exibição de erros ativada, o PHP consegue capturar o erro, 
formatar como um texto HTML e enviar com sucesso como resposta. 
Para o servidor, o script PHP rodou e produziu uma saída, então ele considera a requisição um sucesso.

O que foi retornado no corpo da resposta HTTP?
Foi retornada a mensagem de erro detalhada do PHP, formatada em HTML. 
Informa o tipo de erro (Parse error), o caminho completo do arquivo no servidor e a linha exata onde ocorreu.

Em qual situação essa configuração seria útil?
Essa configuração é útil em um ambiente de desenvolvimento, 
permite que o programador veja os erros instantaneamente no navegador, o que acelera muito o processo de debug.

Em qual situação ela jamais deveria ser utilizada?
Jamais deve ser utilizada em um ambiente de produção. 
A mensagem de erro expõe informações sensíveis que um invasor pode usar para explorar vulnerabilidades. 
Além disso, exibir erros de código para um usuário final é confuso, 
quebra o layout da página e passa uma imagem de amadorismo e instabilidade.
